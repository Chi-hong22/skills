# Skills 项目

本项目用于存储与创建个人 Skill 文件。

## 项目架构

```
skills/
├── .cursor/skills/      # 正在使用的 Cursor Skills（从其他地方安装/引用的）
└── skills/              # 自己创建的原创 Skills
```

## 目录说明

- `.cursor/skills/` - 正在使用中的技能文件，包含从 Cursor 技能市场或其他来源安装的技能
- `skills/` - 自己创建的原创技能文件，用于个人项目或分享

## 文档更新规范

**重要提醒**：任何功能、架构、写法更新必须在工作结束后更新相关目录的子文档。

### 文档更新流程

1. 每个文件夹中都有一个极简的架构说明文件（README.md），包含：
   - 3行以内的架构说明
   - 每个文件的名字、地位、功能
   - 文件开头声明：一旦所属的文件夹有所变化，请更新本文档

2. 每个代码文件的开头需要三行极简注释（除非已存在详细注释或元数据）：
   - **Input（依赖）**：依赖外部的什么
   - **Output（输出）**：对外提供什么
   - **Pos（地位）**：在系统局部的地位是什么
   - 并注明：一旦文件被更新，务必更新开头注释及所属文件夹的 README.md

3. 特殊情况：
   - SKILL.md 文件如果包含元数据（frontmatter），则不需要添加三行极简注释
   - 已有详细文档字符串的文件（如 Python 脚本），则不需要额外注释
